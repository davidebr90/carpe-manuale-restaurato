Codifica / Decodifica / Modifica dei file .E1, .EC, .AG ed .XML

Strumento interattivo per decodificare un file .E1 di Carpe in XML, modificarlo e riencoderlo. Tutto offline, tutto nel browser. Algoritmo dedotto dall'analisi di Sub-Est.dll → E1READ.

Algoritmo confermato. Il file .E1 di Carpe è cifrato con una semplice sottrazione modulo 256 usando una chiave ASCII di 40 byte ciclica. Niente AES, niente compressione, niente Crypto API. La chiave è hardcoded nell'esportazione E1READ di Sub-Est.dll e qui sotto trovi un decoder/encoder funzionante che gira interamente nel tuo browser.

1. Carica il file

Trascina qui sotto un file .E1 (cifrato), .EC (in chiaro), .AG (anagrafica AGO) oppure .XML. Il tool riconosce automaticamente il tipo e lo decodifica/visualizza nel pannello successivo. Tutto avviene nel tuo browser: nessun upload remoto, nessuna traccia esterna.

Trascina qui il tuo file oppure clicca per sfogliare

L'algoritmo, in dettaglio


Il file .E1 è cifrato con una sottrazione modulo 256 usando una chiave ASCII di 40 byte ciclica. Niente AES, niente compressione. Algoritmo:



plain[i] = ( e1[i] − key[i mod 40] ) mod 256


e1[i]    = ( plain[i] + key[i mod 40] ) mod 256




  
Chiave (ASCII, 40 byte) — estratta da Sub-Est.dll → E1READ

  …
  
Stessa chiave in esadecimale

  …




Implementazione in 3 linguaggi




  

    
    
    
    
    
  

  
KEY = b"1E1eBa59$eCf8$4F7d$9bD2$e7EEbba02393$12f"

def decode_e1(data: bytes) -> bytes:
    return bytes((b - KEY[i % 40]) & 0xFF
                 for i, b in enumerate(data))

def encode_e1(data: bytes) -> bytes:
    return bytes((b + KEY[i % 40]) & 0xFF
                 for i, b in enumerate(data))

with open("input.e1", "rb") as f:
    blob = f.read()
xml = decode_e1(blob).decode("utf-8")
print(xml)

  
  
  
  







Evidenze dell'analisi statica


    
  
  • Sub-Est.dll esporta E1READ, E1FROME1, ECFROME1, CARICAE1: è il modulo che gestisce davvero la lettura dell'estratto, non ca-r-et.exe che è solo un wrapper.
    • 
  
  • Nel blocco statico di E1READ compaiono in chiaro: \*.E1, E1Read, E1READ, la costante 000000040 (= lunghezza chiave) e la stringa di 40 caratteri della chiave.
    • 
  
  • Applicando la chiave con sottrazione modulo 256 si ottiene XML valido sia sul file utente sia su un secondo campione indipendente preso dalla cartella SAVEDATA di Carpe.
    • 
  
  • Il primo apostrofo della stringa nel binario è solo un delimitatore, non parte della chiave (verificato per esclusione: con apostrofo l'XML è frammentato).
    •

Struttura del file XML decodificato



L'XML decodificato ha radice <RispostaEstrattoConto> e contiene tipicamente queste sezioni principali (i nomi esatti possono variare):



    
  
  • Anagrafica assicurato: cognome, nome, codice fiscale, data e luogo di nascita, indirizzo di residenza.
    • 
  
  • Tabelle codici: codici contributivi, codici gestione, codici qualifiche, ecc.
    • 
  
  • Periodi contributivi: lista di record con data inizio/fine, settimane utili, retribuzione, codice azienda, codice contribuzione, gestione di accredito.
    • 
  
  • Anagrafica datori di lavoro: codice azienda 10 cifre + ragione sociale.
    • 
  
  • Note tecniche e flag: campi di servizio per la coerenza interna del calcolo Carpe.
    • 





  
  

    Suggerimento. Decodifica una volta il tuo .E1, salva l'XML, e usalo come master. Le modifiche che vuoi simulare (retribuzione futura, periodi mancanti, riscatti) le fai sul file XML editandolo nel tool qui sopra. Poi rigeneri il .E1 con il bottone «Scarica .E1 modificato». Il file E1 risultante può essere ricaricato in Carpe per fare ulteriori calcoli.
